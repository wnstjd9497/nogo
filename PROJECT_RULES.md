# PROJECT_RULES

이 문서는 이 레포에서 앞으로 작업할 때 항상 지킬 운영 규칙입니다.

## A. 공통 작업 순서(항상 수행)
1. **기존 파일 먼저 읽기**
   - 요청과 관련된 기존 파일을 먼저 읽고 현재 상태를 파악합니다.
2. **부족한 점 5개를 '제안'으로 먼저 작성**
   - 실제 수정 전에, 개선 제안 5개를 먼저 정리해 공유합니다.
3. **실제 변경은 PR로 진행**
   - 변경사항은 반드시 커밋 후 PR로 제출합니다.
   - **자동 머지는 하지 않습니다.**

## B. 스크립트 파일 수정 시 필수 점검
스크립트(`scripts/*.md` 등) 파일을 만들거나 수정할 때 아래를 반드시 확인합니다.

1. 더 쉬운 한국어로 다듬기
2. 환자 질문(FAQ) 최소 8개 점검
3. 7문장 암기버전 포함 여부 점검
4. 과장/허언/확정 표현 줄이기
5. 레드플래그(바로 연락해야 하는 증상) 안내 문장 포함

## C. PR 설명 필수 형식
모든 PR 설명에는 아래 3가지를 반드시 포함합니다.

1. **무엇을 바꿨는지**
2. **왜 바꿨는지**
3. **내가 확인할 것 3개**

## 현재 기준 제안(기존 파일 검토 결과)
아래 제안은 `templates/script-template.md`와 `scripts/lifting-doublechin.md`를 읽고 정리한 개선 포인트입니다.

1. `scripts/lifting-doublechin.md`에 FAQ가 현재 4개라서, 최소 8개 기준에 맞게 확장 필요
2. `scripts/lifting-doublechin.md`에 레드플래그를 별도 소제목으로 분리하면 현장 가독성 개선 가능
3. `templates/script-template.md`에 "스크립트 작성 전 체크리스트"를 추가하면 초심자 활용성이 높아짐
4. 두 문서 모두 문장 길이가 긴 부분을 더 짧게 끊으면 환자 설명 시 전달력이 좋아짐
5. 공통 표현(예: "개인차가 있습니다", "재평가 후 결정")을 별도 문구 라이브러리로 모으면 일관성 유지에 유리
